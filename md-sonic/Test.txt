1. Install Dependencies
jdk1.8
maven

Installation steps:
tar -xzf apache-maven-3.9.6-bin.tar.gz
tar -xzf jdk-8u401-linux-x64.tar.gz
mv jdk1.8.0_401/ /usr/local/java
mv apache-maven-3.9.6 /usr/local/apache-maven-3.9.6
vim /etc/profile
Add the following lines:
export JAVA_HOME=/usr/local/java
export JRE_HOME=${JAVA_HOME}/jre
export M2_HOME=/usr/local/apache-maven-3.9.6
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib:${M2_HOME}/lib
export PATH=.:${JAVA_HOME}/bin:$PATH:${M2_HOME}/bin
Save and exit.
source /etc/profile

Edit /usr/local/apache-maven-3.9.6/conf/settings.xml
Replace the mirror section as follows (switch to Aliyun mirror):
<mirror>
    <id>aliyunmaven</id>
    <mirrorOf>*</mirrorOf>
    <name>aliyun maven</name>
    <url>https://maven.aliyun.com/repository/public</url>
</mirror>
Save and exit.

2. Build the Code
cd md-sonic
mvn package -DskipTests

3. Micro Benchmarks (2PC, 10,000 iterations)
cd demos/bench
make move_2PC
make runRelu_2PC
cat server1/log.txt   (check test results)
make clean

4. 4PC, 8PC, 16PC, 32PC, 64PC
You can change the make command to move_%PC and runRelu_%PC.
When the number of parties is large, 10,000 iterations may take a long time.
It is recommended to run inside Docker and start from smaller party numbers.
