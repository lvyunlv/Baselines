1. Install Dependencies (Ubuntu 22.04)
CMake (>= 3.16)    apt install cmake
g++ (Support C++20)    apt install build-essential
Eigen3    apt install libeigen3-dev

2. Build the Code
cmake -DCMAKE_BUILD_TYPE=Release -S . -B build/
cmake --build build/ --config Release --target all -j

3. Code Testing --- Micro Benchmarks (relu sigmoid tanh gelu softmax)
## Single-machine test: bash run-microbenchmarks-local.sh
In the output, find the line corresponding to the module, e.g., "relu: 13 ms, 332.031 KB"

## Two-machine test:
On Server1 (10.0.0.1): bash run-microbenchmarks-remote.sh 0 10.0.0.2
On Server2 (10.0.0.2): bash run-microbenchmarks-remote.sh 1 10.0.0.1

4. Code Testing --- Benchmarks (Bert-Base, 128 input tokens)
## Single-machine test: bash run-benchmarks-local.sh
In the output:
"bert: 65093 ms, 1.49624e+06 KB" means the total cost;
"nonlinear: 14842 ms, 553140 KB" means the cost of the nonlinear part;
The difference is the cost of the linear part.

## Two-machine test:
On Server1 (10.0.0.1): bash run-benchmarks-remote.sh 0 10.0.0.2
On Server2 (10.0.0.2): bash run-benchmarks-remote.sh 1 10.0.0.1

5. Parallel Execution
You can modify OMP_NUM_THREADS=4 in the .sh scripts.

6. Change Input Size
In the corresponding cpp file under the microbenchmarks folder, modify the value of n and rebuild.

7. Fixed-point Precision
The cost of gelu, tanh, softmax, and sigmoid depends on the fixed-point precision f.
Currently f=16. To change it, edit the corresponding cpp file in the microbenchmarks folder, update f, and rebuild.
